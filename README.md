# Проект: Приют-питомник для собак

Данный проект представляет собой веб-приложение для управления приютом-питомником. Система включает три основных функциональных модуля:

1. **Работа с питомцами** — управление карточками собак.
2. **Личный кабинет пользователя** — регистрация, аутентификация и управление профилем.
3. **Работа с отзывами** — создание и просмотр отзывов о питомцах.

---

## Технологический стек

- **Backend:** Django, Django REST Framework (DRF)
- **База данных:** PostgreSQL / SQLite (в зависимости от настроек)
- **Кэширование:** Redis
- **Виртуальное окружение:** `venv`

---

## Установка и запуск проекта

Следуйте инструкциям ниже, чтобы развернуть проект локально.

### 1. Клонирование репозитория
    git clone <https://github.com/BetepIvan/Web524ShelterCBV>
    cd <Web524ShelterCBV>

### 2. Настройка виртуального окружения
    python -m venv venv
    source venv/bin/activate   # Для Linux/macOS
    venv\Scripts\activate      # Для Windows

### 3. Установка зависимостей
```bash
    pip install -r requirements.txt
```

### 4. Настройка переменных окружения
Создайте файл ```.env``` в корне проекта и заполните его данными по образцу из ```.env_sample``` (секретный ключ, настройки базы данных и т.д.).

### 5. Работа с базой данных
Создайте базу данных и примените миграции:

```bash
# Создание базы данных (если используется кастомная команда)
python manage.py ccdb

# Создание файлов миграций
python manage.py makemigrations

# Применение миграций
python manage.py migrate
```

### 6. Заполнение базы данных тестовыми данными
Создайте пользователей с разными уровнями доступа и загрузите фикстуры:

```bash
# Создание пользователей (админ, модератор, обычные пользователи)
python manage.py ссsu

# Загрузка фикстур с данными о собаках и отзывах
python manage.py loaddata dogs_fixture.json
python manage.py loaddata reviews_fixture.json
```

### 7. Запуск Redis
Redis необходим для работы кэширования. Откройте новое окно терминала и выполните:

```bash
redis-server
```

### 8. Запуск сервера разработки
Вернитесь в основной терминал (или откройте новое окно) и запустите сервер:

```bash
python manage.py runserver
```
После успешного запуска проект будет доступен по адресу: http://127.0.0.1:8000/


## Модели базы данных
### Dogs (Собаки)
| Поле | Тип | Описание |
|------|-----|----------|
| `name` | CharField | Кличка собаки |
| `breed` | ForeignKey | Порода |
| `photo` | ImageField | Фотография |
| `birth_date` | DateField | Дата рождения |
| `is_active` | BooleanField | Активен ли питомец (находится ли в приюте) |
| `owner` | ForeignKey | Связь с моделью `User` (владелец) |
| `views` | IntegerField | Количество просмотров |
### Review (Отзывы)

| Поле | Тип | Описание |
|------|-----|----------|
| `title` | CharField | Заголовок отзыва |
| `slug` | SlugField | Уникальный идентификатор для URL |
| `content` | TextField | Содержимое отзыва |
| `created` | DateTimeField | Дата и время создания |
| `sign_of_review` | BooleanField | Активен ли отзыв (модерация) |
| `author` | ForeignKey | Связь с моделью `User` (автор) |
| `dog` | ForeignKey | Связь с моделью `Dog` |

### User (Пользователь)

*Расширенная модель пользователя.*

| Поле | Тип | Описание |
|------|-----|----------|
| `email` | EmailField | Электронная почта (логин) |
| `first_name` | CharField | Имя |
| `last_name` | CharField | Фамилия |
| `avatar` | ImageField | Аватар профиля |
| `phone` | CharField | Номер телефона |
| `telegram` | CharField | Аккаунт в Telegram |
| `max_messenger` | CharField | Аккаунт в MAX |
| `is_active` | BooleanField | Состояние учетной записи |

### Роли пользователей

| Роль       | Описание |
|------------|----------|
| `Admin`	     | Полный доступ ко всем функциям системы|
| `Moderator`	 | Расширенные права: просмотр списка пользователей, модерация контента|
| `User`       | Обычный авторизованный пользователь: создание карточек, отзывов, редактирование профиля|

### Права доступа (Permissions)
- Реализованы кастомные разрешения для пользователя с ролью MODERATOR
- Смотреть списки пород, собак и отзывов могут все пользователи
- Смотреть подробную информацию о породе, собаке и отзыве могут все пользователи
- Создавать, Изменять, Удалять карточку могут только авторизованные пользователи
- Список пользователей могут видеть Администратор или Модератор
- Регистрироваться на ресурсе может любой пользователь
- Видеть подробную информацию о пользователе могут все авторизованные пользователи
- Изменять информацию о пользователе может только сам пользователь
- Удалять пользователя может только Администратор

### Дополнительные функции
Пагинация: Реализована пагинация для списков сущностей (собаки, отзывы, пользователи) — 3 элемента на страницу.  
Поиск: Реализован поиск по породам, кличкам собак и отзывам (общий поиск по ключевым словам).
